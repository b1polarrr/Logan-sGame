# Poker_AA 开发约定

本文档记录本项目的协作方式、编码规范与当前进度。  
**新开 Cursor 对话时，请 @ 本文件**，以便 AI 延续相同约定。

---

## 一、与 AI 协作方式

### 1.1 手搓优先，增量修改

- 采用**导师式、一步一步**推进，每次只改一小段，讲清楚「改哪个文件、哪一段、为什么」。
- **禁止**默认「整文件替换」；除非用户明确要求，否则在现有代码基础上增量修改。
- 每完成一小步，用户应 **Build / 编译** 确认无报错后再继续下一步。

### 1.2 不写 import 步骤

- 指导代码时**不再单独列出「添加 import」的步骤**。
- 由开发者在 IDEA 中使用 **Alt + Enter** 自动导入缺失的包。

### 1.3 沟通语言

- 与用户交流使用**简体中文**。

## 二、命名规范

### 2.1 变量与参数：尽量全称

| 允许 | 禁止（示例） |
|------|----------------|
| `gameManager` | `gm` |
| `playerSession` | `ps` |
| `roomId` | `rid` |
| `seatIndex` | `si` |
| `responseBuilder` | `rb` |
| `snapshotBytes` | 无意义缩写 |

### 2.2 例外：Netty 惯例

- `ChannelHandlerContext` 的参数名可保留 **`ctx`**（业界通用）。

### 2.3 单例方法名（本项目现状）

以下类使用 **`getINSTANCE()`**（非 `getInstance()`），新增代码请与现有风格保持一致：

- `SessionManager.getINSTANCE()`
- `SnapshotBroadcaster.getINSTANCE()`

以下类使用标准 **`getInstance()`**：

- `RoomRouter.getInstance()`

---

## 三、代码风格

- **最小改动**：只改与当前任务相关的代码，不顺手重构无关部分。
- **匹配现有风格**：包名、缩进、注释密度与周边文件保持一致。
- **注释**：只解释非显而易见的业务逻辑；避免废话注释。
- **不主动提交 Git**：除非用户明确要求 commit。

---

## 四、项目架构速览

```
浏览器 (Vue3 SPA / test.html 联调页)
    ↕ WebSocket (ws://localhost:8888/ws 或 Vite 代理 /ws) + Protobuf 二进制帧
PokerNettyServer → PokerGameHandler → RoomRouter → GameManager
                              ↓
                    SessionManager（连接 ↔ 房间 ↔ 座位）
                              ↓
                    SnapshotBroadcaster（ServerMessage 广播）
```

| 包 / 目录 | 职责 |
|-----------|------|
| `engine/` | 德州扑克引擎：牌型、牌桌、下注、比牌 |
| `network/` | Netty 网关、会话、路由、快照广播 |
| `src/main/proto/` | Protobuf 协议定义 |
| `frontend/` | Vue3 + Vite 正式前端（大厅 + 牌桌） |
| `test.html` | 联调页（阶段 1/2 备用） |

## 七、联调备忘

- 服务端入口：`PokerNettyServer.main()`，端口 **8888**，路径 **`/ws`**
- 正式前端：`frontend/` → `npm run dev`（5173），WS 走 Vite 代理 `/ws`
- 联调流程：连接 → 创建/进入房间 → 坐下（两人）→ 自动开局 → 游戏动作
- 游戏动作的 `seatIndex` 以 **`PlayerSession.getSeatIndex()`** 为准，不信任前端传来的座位号（SIT_DOWN 选座除外）
